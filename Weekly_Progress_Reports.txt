1. WORK CARRIED OUT BY THE TEAM
────────────────────────────────

  • Developed the PPO Reinforcement Learning signal control system (rl_model.py):
    - Custom Gymnasium environment (TrafficIntersectionEnv) simulating a
      4-way intersection with Discrete(2) action space (NS_GREEN / EW_GREEN)
      and Box(0,100,shape=(4,)) observation space for queue lengths.
    - Reward function: negative sum of all queues + variance penalty for fairness.
    - Trained the PPO model with MlpPolicy for 5,000 timesteps and saved
      weights to ppo_traffic_model.zip (139 KB).

  • Integrated the RL model into the WebSocket telemetry loop with the
    following signal control priority logic:
      1. If AI paused → use manual phase (operator override)
      2. If emergency active → force green on dominant lane direction
      3. Otherwise → PPO predict_action() from live lane queues

  • Built the simulation engine (simulation_api.py) for live junction analysis:
    - 10 Bangalore junctions mapped with GPS coordinates and snapshot IDs.
    - Combines TomTom traffic + OpenWeatherMap weather + DB junction snapshots.
    - POST /api/simulate and GET /api/simulate/autofill endpoints.
    - GET /api/junctions returns all supported junction IDs and names.

  • Developed the Command Center REST API (command_api.py):
    - GET /api/command/status: live traffic snapshot with congestion labels
    - POST /api/command/control: operator overrides (mode, emergency, force_phase)
    - GET /api/command/logs: signal_logs retrieval from Supabase
    - All operator overrides are logged to signal_logs for audit trail.

  • Implemented the RAG-powered AI chatbot (agent.py):
