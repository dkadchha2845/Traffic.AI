Slide 2: Overview
• Goal: Optimize urban mobility in Bangalore using artificial intelligence.
• Core Technologies: Computer Vision, Reinforcement Learning, Time-Series Forecasting.
• Key Feature: Real-time adaptive signal control that overrides traditional fixed timers.
• Scope: Active monitoring and control across 11 major Bangalore traffic zones.

================================================================================

Slide 3: Abstract
• Urban congestion causes significant economic and environmental losses globally.
• Traditional fixed-timer signals fail to adapt to live, unpredictable traffic conditions.
• Our solution, Traffic.AI, proposes a multi-agent AI system (YOLOv8 + PPO RL).
• Achieves real-time monitoring and adaptive delay reduction via a full-stack dashboard.

================================================================================

Slide 4: Problem Statement
• Fixed-timer signals lead to uneven congestion and wasted green phases on empty roads.
• Traffic authorities lack centralized, real-time situational awareness across multiple junctions.
• No automated priority routing exists for emergency vehicles (ambulances, fire engines).
• Inability to predict traffic build-ups and preemptively manage traffic flow.

================================================================================

Slide 5: Introduction
• Bangalore suffers from severe traffic bottlenecks at key junctions (e.g., Silk Board, Hebbal).
• Internet of Things (IoT) and AI advancements enable dynamic, data-driven traffic control.
• Our project transitions traffic management from reactive to proactive and predictive.
• Integrates live CCTV video feeds, TomTom traffic data, and real-time weather APIs.

================================================================================

Slide 6: Social Impact
• Reduces travel time and fuel consumption for daily commuters.
• Enables faster emergency vehicle response times, potentially saving lives.
• Lowers vehicular carbon emissions by significantly reducing idling times at red lights.
• Improves overall productivity and quality of life for urban residents.

Slide 8: Functional Requirements
• System must detect, classify, and count vehicles from live CCTV streams in real-time.
• Reinforcement learning agent must dynamically adjust signal phases based on queue lengths.
• Dashboard must display live telemetry (10 frames per second) across all 11 zones.
• System must support manual operator overrides and automatic emergency green-waves.
• AI Chatbot must answer natural language queries using live and historical traffic data.

================================================================================

Slide 9: Non-Functional Requirements
• Latency: Sub-second latency for WebSocket telemetry streaming and AI decisions.
• Reliability: Graceful degradation (fallback to local baselines if TomTom API fails).
• Security: Secure access via Supabase JWT authentication and role-based access.
• Scalability: Ability to process multi-camera feeds concurrently without UI blocking.
• Usability: Cross-platform, highly responsive React UI for command center operators.

================================================================================

Slide 10: System Requirements
• Hardware:
  - Processor: Intel Core i5 / AMD Ryzen 5 or higher
  - Memory: Minimum 8GB RAM (16GB recommended for concurrent multi-camera tracking)
  - GPU: Optional but recommended for stable YOLOv8 inference at 30+ FPS
• Software:
  - OS: Windows 10/11, Linux, or macOS
  - Backend environment: Python 3.10+, Uvicorn
  - Frontend environment: Node.js 18+, Modern Web Browser (Chrome/Edge/Firefox)
  - Database: Supabase (Managed PostgreSQL)

================================================================================

Slide 11: Design & Architecture
• Frontend Presentation: React 18, TailwindCSS, Recharts, Three.js (Digital Twin).
• Backend Application: FastAPI, WebSocket for bi-directional live telemetry streaming.
• AI/ML Layer: YOLOv8 (Vision), Stable Baselines3 PPO (RL), LangChain (RAG).
• Data Persistence: Supabase (Relational tables + Vector store for embeddings).
• External Services: TomTom Traffic Flow API, OpenWeatherMap API.

================================================================================

Slide 12: Methodology
• Step 1 - Data Collection: Integration of TomTom APIs and sample CCTV video feeds.
• Step 2 - Vision Model: Deploying YOLOv8n to extract lane-wise queue counts via spatial mapping.
• Step 3 - RL Environment: Building `TrafficIntersectionEnv` with a custom fairness reward function.
• Step 4 - Full-Stack Dev: Developing the FastAPI backend routing and React frontend dashboard.
• Step 5 - Integration: Connecting AI models via 10 FPS WebSocket telemetry loops.
• Step 6 - Testing: Validating fallback mechanisms, API latency, and visual rendering accuracy.

================================================================================

Slide 13: Technologies Used
• Web Technologies: React, TypeScript, Vite, Tailwind CSS, Framer Motion.
• Backend Technologies: Python, FastAPI, WebSockets, Pydantic.
• AI/ML Libraries: Ultralytics (YOLO), Stable Baselines3, Gymnasium, LangChain, pmdarima.
• Database & Hosting: Supabase (PostgreSQL), Vercel (Frontend deployment).
• Visual Utilities: React-Leaflet (Maps), Three.js (3D Visualization), ReportLab (PDFs).

================================================================================

Slide 14: AI Models
• YOLOv8n (Vision): Fast, lightweight object detection tracking cars, bikes, buses, and trucks.
• Proximal Policy Optimization (PPO): RL agent that learns optimal signal policies; reward function minimizes total delay and queue variance.
• AutoARIMA (Forecasting): Time-series model providing 30-minute congestion forecasting based on historical density data.
• RAG Chatbot: Combines LangChain with Supabase Vector Store for context-aware, natural language operator assistance.
